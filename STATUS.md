# E2E test status

State of the kvikdos test suite + QEMU smoke pipeline on the `build`
branch. Both VC builds are green end-to-end.

## What's in the tree

* `vendor/kvikdos` — submodule pinned to the `improvements` tip
  (in-place AH=4B AL=00 child spawn, command-tail NUL preservation,
  EXE-alloc fallback, .exe BSS zeroing, MCB program-name stamping,
  AH=06 two-call extended-key delivery, AH=44 AL=0F / AL=0D / INT 2Fh
  stubs — all needed to host VC 4.99.09's overlay).
* `tests/test_common.h`, `tests/fixtures/`, 28 `tests/test_*.c` —
  shared harness plus per-feature test groups. `setup_fixtures()`
  drops `KVIKPROG.OVL` + `VC.OVL` and the captured 4.99 INI into the
  per-test temp dir when `VC_OVL_PATH` is set; `test_is_vc_499()`
  exposes the version split to test code.
* `tests/test_volkov_e2e.sh`, `tests/screen_expect.py` — QEMU smoke
  test. Takes `<version>` and uses `build/<version>/VC.COM` (+ `VC.OVL`
  for 4.99.09). Reuses the floppy image `build.sh` caches under
  `build/.cache/`.
* `Makefile` — top-level. Targets:
  * `make build-tests` — assembles all kvikdos test binaries.
  * `make test-kvikdos-4.05` / `make test-kvikdos-4.99.09` — the
    4.99 group excludes editor tests (`test_editor`,
    `test_deep_editors`, `test_gaps_editors`) and sets `VC_OVL_PATH`.
  * `make test-qemu-{4.05,4.99.09}` — QEMU smoke per version.
  * `make test-{4.05,4.99.09}` / `make test` — combined.
* `.github/workflows/e2e.yml` — CI matrix per VC version: build →
  build kvikdos test binaries → kvikdos suite → QEMU e2e → upload
  artifacts.

## Per-version status

### VC 4.05

* **kvikdos:** ✅ all test groups pass against `build/4.05/VC.COM`.
* **QEMU smoke:** ✅ passes.

### VC 4.99.09

* **kvikdos:** ✅ all base test groups pass against
  `build/4.99.09/VC.COM` + `VC.OVL`. (Editor groups don't apply —
  4.99 has no built-in editor; F4 invokes whatever `VC.EXT`
  configures.)

  The path here required several kvikdos changes (in-place AH=4B
  AL=00 child spawn for the OVL, command-tail NUL preservation,
  EXE-alloc fallback, BSS zeroing, MCB program name stamp, AH=06
  two-call extended-key delivery, INT 2Fh stubs) plus harness/test
  changes:
  * `tests/test_common.h` exposes `test_is_vc_499()` (proxy:
    `VC_OVL_PATH` is set), and `setup_fixtures()` drops both
    `KVIKPROG.OVL` and `VC.OVL` next to the program plus the
    captured 4.99 INI (`tests/fixtures/VC.4.99.09.INI`, generated by
    `tests/capture_vcini.sh`) so VC starts in a known panel state
    instead of the "Can not read setup file" dialog.
  * Per-test branches use `test_is_vc_499()` to skip 4.05-only
    bindings or to substitute equivalent flows. The most recurring
    differences:
    - 4.99 routes Gray-* / Numpad-+ / Numpad-- scancodes to the
      command line instead of binding them as panel commands.
    - 4.99's Left/Right dropdowns add Description / Quick view
      items above Tree, so position-based DOWN×N walks land on the
      wrong row — tests now use letter hotkeys (`type_string("t")`).
    - 4.99 opens an extra "View the file:" / "Edit the file:"
      confirmation dialog on F3/F4; tests Enter-confirm if it
      appears.
    - 4.99's help viewer doesn't expose F7 search or the Tab+Enter
      hyperlink walk the 4.05 tests rely on; `ESC`-after-link
      tears VC down. Tests skip those subtests under 4.99.
    - 4.99 has no internal editor — `test_editor`, `test_deep_editors`,
      `test_gaps_editors` are excluded from `TEST_GROUPS_4_99_09` in
      the Makefile.
* **QEMU smoke:** ✅ passes 5/5.

## Open items
